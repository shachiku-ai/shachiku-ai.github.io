Command Line Assistants (CLI) Integration Guide

The Shachiku backend has a built-in mechanism to interact directly with command-line AI assistants (CLI). By configuring a CLI tool, you can allow Shachiku to invoke authorized terminal AI assistants locally. This broadens its use cases and meets highly customized high-end requirements.

Supported CLI Tools and Installation Methods

Currently, we support the following command-line assistants. You must first install and configure the corresponding CLI tools on your operating system to ensure they run normally in the local terminal.

1. Claude Code

A powerful command-line tool officially launched by Anthropic to interact directly with the Claude LLM system.

  • Official Website / Documentation: Claude Code Overview
  • Installation Method: Ensure Node.js is installed in your environment, then run the following command to install it globally: 
    npm install -g @anthropic-ai/claude-code
  • Configuration: Once installed, run claude in your terminal and complete Anthropic's billing authorization based on OAuth.

2. Gemini CLI

A terminal tool provided by Google for interacting with the Gemini architecture.

  • Open-source Website / Toolkit: generative-ai-cli (or similar open-source implementations)
  • Installation Method: 
    npm install -g @google/generative-ai-cli
    (Note: Refer to the repository above for exact terminal commands, or use popular Python-based CLI alternatives).

3. Other Compatible CLIs

Such as codexcli. If you use other open protocol-based CLI tools relying on classic standard IO, make sure they are installed and added to the system's environment variables (PATH), so they can be securely and globally invoked by the system terminal.

System Configuration

  1. Make sure the above installation is successful and the command can be invoked smoothly in any directory configuration (for example, typing claude -h returns the help prompt).
  2. Go to Shachiku's Provider settings page section and select the Command Line (CLI) as your integrated LLM solution.
  3. Specify the corresponding CLI command name and extra startup parameter variables as required by your framework environment, then hit save!
